```java
为什么不用联表查询？
select * from tag
join tag_post on tag_post.tag_id=tag.id
join post on tag_post.post_id=post.id
where tag.tag='mysql';


单表
select * from tag where tag='mysql';
select * from tag_post where tag_id=1234;
select * from post where id in(123,456,567,9989,
                               
                               
1. 单表利于后续维护，如果后续表结构变动join 需要变更，单表查询利于维护变更。
2. 单表查询代码复用行高，便于组装数据。
3. 效率问题，如果数量增加，多表查询是笛卡尔乘积的方式，检索数据是几何上升的。
4. 缓存利用率更高，比如上面查询中的tag是不常变动的数据，缓存下来，每次查询就可以跳过第一条查询语句。而关联查询，任何一张表的数据变动都会引起缓存结果的失效，缓存利用率不会很高。
5. 其他，分库分表 join 查询不利于分库分表。跨库join表现不良
```
